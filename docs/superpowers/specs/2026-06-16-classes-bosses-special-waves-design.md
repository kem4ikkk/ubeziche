# Дизайн: классы игрока (4.12) + боссы/спецволны/C4 (4.13)

> Проект «Убежище» (Godot 4.6.3, GDScript). Приведение базового цикла к
> CS:NZ «New Zombie Shelter». Дата: 2026-06-16. Модель: Claude Opus 4.8.

## 1. Контекст и цель

Сейчас в игре есть плоское дерево навыков (3 ветки: Добыча/Бой/Инженер, по
3 уровня; очки 3 на старте + 1/ночь) **без выбора класса**, мини-босс
«Джаггернаут» и масштабирование числа зомби по ночам. Спецзомби, настоящего
босса, спецволн и сигнатурных способностей классов нет.

Цель — добавить **классы игрока с идентичностью (4.12)** и **боссов/спецволны
с C4 (4.13)**, как в оригинале, оставаясь играбельным в одиночку.

Источники по оригиналу: [CSO Wiki — New Zombie Shelter](https://cso.fandom.com/wiki/New_Zombie_Shelter),
[Steam Guide (co-op)](https://steamcommunity.com/sharedfiles/filedetails/?id=484424330).
Находка: в оригинале ветки технически НЕ заблокированы, но гайды советуют
качать одну («choose a path and stay on it») — отсюда модель «строгий класс +
урезанный кросс-доступ».

## 2. Решения автора (подтверждено в брейнсторме 2026-06-16)

- **Классы:** строгий выбор класса на старте (вариант «как оригинал»), НО можно
  частично качать чужие ветки.
- **C4:** и AoE по зомби, и снос проходов — но снос только в **определённых
  размеченных местах**, обычные постройки/базу сносить нельзя.
- **Охват 4.13:** полный набор (Крикун + Взрывной + новый Босс с HP-баром +
  теги спецволн).

## 3. Архитектура (где жить коду)

- **Состояние класса/навыков** — расширяем `InventorySystem` (там уже
  `skill_points`, `gather/combat/engineer_level`, `SKILL_MAX`, `upgrade_skill`,
  на них завязаны `resource_pickup`/`player`/меню). Отдельный autoload сейчас НЕ
  выделяем (крупный несвязанный рефактор с риском для capture-тестов; YAGNI).
- **Новые системы** — отдельными модулями/сценами, чтобы не раздувать файлы:
  сигнатурные способности (обработчик в `player.gd` + сцены эффектов), C4
  (сцена+скрипт), спецзомби (скрипты на базе `zombie.gd` + сцены), босс
  (скрипт+сцена), HP-бар босса (Control), спецволны (логика в `wave_manager.gd`).

## 4. Классы и навыки (4.12)

**Выбор класса на старте забега** — панель «Выберите класс» (3 кнопки +
описания). В capture-режиме класс задаётся программно (по умолчанию для тестов
построек — Инженер; в тестах перебираем все три).

| Класс | Стат-навык (ур. 1→3) | Сигнатурная способность (только свой класс) |
|---|---|---|
| **Боец** (combat) | `combat_level`: +урон ближнего (×4, как сейчас) **и** +макс HP (+15/ур.) | **Авиаудар** |
| **Добытчик** (gather) | `gather_level`: +1 ресурс/удар **и** +лимит ресурсов (+20/ур.) | **Костёр** |
| **Инженер** (engineer) | `engineer_level`: +HP ремонта **и** +урон турелей | **C4** |

- **Очки:** 3 на старте + 1/ночь (без изменений).
- **Кросс-доступ:** очко можно вложить в чужую ветку, но её потолок = **1**
  (тюнится константой), сигнатурную способность чужого класса не открыть.
- **Сигнатурная способность** = отдельный узел своей ветки: стоит 1 очко,
  требует совпадения класса и уровня своей ветки ≥1.
- **Активация способности — клавиша F** (контекстно по классу; `F` свободна,
  ремонт ушёл на удар топором ещё в 4.21).
- **+макс HP бойца:** при повышении `combat_level` поднимаем `max_health`
  игрока (через HealthComponent) и подлечиваем на дельту.
- **+лимит ресурсов добытчика:** `RESOURCE_CAP` становится динамическим —
  базовый 40 + 20×`gather_level` для класса Добытчик (иначе базовый).
- **+урон турелей инженера:** игрок один, поэтому турель читает глобальный
  `InventorySystem.engineer_level` и множит свой урон (напр. +10%/ур.).

**UI меню навыков (N):** к 3 строкам веток добавляется строка сигнатурной
способности (для своего класса — кнопка [Открыть]/[Готово]; для чужого скрыта)
и подпись текущего класса вверху. Чужие ветки показывают потолок `[x/1]`.

## 5. Сигнатурные способности (4.12b)

Общая модель: клавиша **F** → `player._use_class_ability()` ветвится по классу.
У каждой способности — кулдаун (для C4 — расход заряда). HUD-подсказка статуса.

- **Авиаудар (Боец):** F → маркер по лучу прицела на землю → через ~1.5 с серия
  взрывов по area (урон по зомби в радиусе). Кулдаун ~25 с.
- **Костёр (Добытчик):** F → ставит костёр (сцена) у игрока. Пока игрок в радиусе
  — реген HP/с. Лимит — 1 костёр одновременно (новый заменяет старый), горит
  фиксированное время N секунд, затем гаснет. Только Добытчик.
- **C4 (Инженер):** см. §6.

## 6. C4 (4.12b, Инженер)

- **Крафт** в мастерской (E) за ресурсы; носится как заряд (счётчик в HUD).
- **F с зарядом C4:** ставит заряд → таймер ~3 с → взрыв:
  - на **земле/рядом с зомби** → AoE урон по зомби (радиус ~4 м);
  - у **размеченного разрушаемого сегмента** (объекты в группе `blastable` в
    заранее заданных местах) → сегмент разрушается, открывая проход;
  - на **обычной постройке/базе** → только AoE, **снести нельзя** (защита:
    ломаются лишь объекты группы `blastable`).
- Размеченные сегменты заранее стоят на карте (1–2 точки), напр. закрывают
  альтернативный маршрут к залежам.

## 7. Спецзомби (4.13a)

- **Крикун (Screamer):** при первом обнаружении игрока один раз «кричит» →
  просит `WaveManager` доспавнить небольшую группу зомби (один раз). Отдельный
  цвет/метка. Скрипт на базе `zombie.gd` + сцена.
- **Взрывной (Exploder):** при смерти → AoE урон по игроку и постройкам в
  радиусе. Отдельный цвет, короткий телеграф перед взрывом. Скрипт+сцена.

## 8. Босс + спецволны (4.13b)

- **Босс «Колосс»:** приходит каждые **5 ночей** (5,10,…). Большой HP,
  уникальная атака — **слэм по площади** (телеграф → урон игроку и постройкам в
  радиусе). Целится в постройки/игрока (логика на базе `zombie.gd`/`juggernaut.gd`).
  Скрипт+сцена.
- **HP-бар босса:** полоса сверху по центру (Control), появляется на спавне,
  скрывается при смерти. Новые сигналы `EventBus`: `boss_spawned(name, max_hp)`,
  `boss_health_changed(hp, max_hp)`, `boss_defeated()`.
- **Спецволны (теги ночей)** в `WaveManager`:
  - **Босс-ночь** (каждые 5): спавнит Колосса + урезанный обычный счёт.
  - **Орда** (напр. каждые 3, если не босс-ночь): ×2 зомби, быстрее интервал.
  - С определённой ночи обычные волны с шансом подмешивают Крикунов/Взрывных.
  - HUD объявляет тип спецволны на старте (расширить `wave_started` меткой или
    добавить сигнал `special_wave(label)`).

## 9. Под-этапы реализации (каждый — коммит + capture-проверка)

- **4.12a — Классы + UI:** выбор класса на старте, `player_class` в
  `InventorySystem`, потолки веток (своя 3 / чужая 1), стат-эффекты (+HP/+лимит/
  +урон турелей), меню навыков N с классом и строкой способности.
- **4.12b — Сигнатурные способности (вкл. C4):** клавиша F, Авиаудар, Костёр,
  C4 (AoE + снос `blastable`), HUD-статус/счётчик C4, крафт C4 в мастерской,
  размеченные сегменты на карте.
- **4.13a — Спецзомби:** Крикун (доспавн), Взрывной (AoE при смерти), сцены+цвета.
- **4.13b — Босс + спецволны + HP-бар:** Колосс, теги ночей в `WaveManager`,
  HP-бар, сигналы EventBus, HUD-объявления спецволн.

## 10. Критерии приёмки (capture, EXIT=0, чтение last_run.png)

- Выбор класса работает; своя ветка качается до 3, чужая стопорится на 1;
  способность чужого класса не открыть.
- Стат-эффекты применяются: +HP бойца, +лимит добытчика, +урон турелей инженера.
- F-способности: Авиаудар наносит AoE по зомби; Костёр лечит игрока рядом; C4
  наносит AoE и сносит `blastable`, но НЕ обычную постройку.
- Крикун доспавнивает группу один раз; Взрывной при смерти бьёт AoE по игроку/
  постройкам.
- Босс-ночь (5-я) спавнит Колосса с HP-баром; слэм наносит урон по area;
  спецволны (орда/босс) логируются и объявляются в HUD.

## 11. Тюнинг-параметры (числа — предварительные, уточняются на балансе)

- Потолок чужой ветки: 1. Боец +HP/ур.: 15. Добытчик +лимит/ур.: 20. Инженер
  +урон турелей/ур.: +10%.
- Авиаудар: задержка 1.5 с, кулдаун 25 с, радиус/урон — по балансу.
- Костёр: реген HP/с, радиус, лимит 1–2, время горения.
- C4: таймер 3 с, радиус AoE 4 м, урон, стоимость крафта.
- Босс: HP, интервал 5 ночей, урон слэма, радиус, телеграф.
- Орда: ×2 зомби, множитель интервала; ночь старта подмешивания спецзомби.
