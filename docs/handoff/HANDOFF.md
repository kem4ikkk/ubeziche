# Хэндовер проекта «Убежище» (контекст для переноса в Cursor)

Документ, чтобы новый ИИ-ассистент (Cursor) быстро вошёл в проект. Рабочие правила —
в `.cursorrules` / `.cursor/rules/`. Подробный журнал — `docs/PROGRESS.md`.
Полный текст нашего диалога — `docs/handoff/chat-history.md`.

## Что это за игра
3D-выживание с обороной базы (вид от 1-го лица), вдохновлено режимом «Убежище»
(New Zombie Shelter) из CS:NZ. Godot 4.6.3, GDScript. Разработчик — соло-инди,
русскоязычный, ассеты по возможности через ИИ/примитивы (внешних моделей нет).

## Ядро игрового цикла
- Днём — добыча ресурсов (дерево/сталь) ударом ТОПОРА по узлам; ночью — волны зомби.
- Зомби идут разрушать **УБЕЖИЩЕ** (периметр серых стен с проходами в центре карты);
  живого игрока агрят и кусают. **Проигрыш** = HP убежища дошло до 0.
- **Смерть игрока ≠ конец**: режим наблюдателя (свободный полёт) + возрождение 20–30 c
  (+1 c за каждую смерть), затем возрождение у базы с полным HP.
- Постройки и апгрейд тира убежища (тир поднимает и HP убежища, и доступ к тир-2/3
  постройкам). Дерево навыков (клавиша N). Психздоровье («шкала холода»).
- Победа — пережить `victory_waves` волн → эвакуация (добежать до зоны за время).

## Автозагрузки
- `InventorySystem` (`scripts/inventory_system.gd`) — ресурсы, деньги, тир, дерево
  навыков (`TREE`, `SKILLS`, `skill_levels`, `player_class`), флаги способностей/инструментов.
- `EventBus` — глобальные сигналы (урон постройке, босс, эвакуация, `shelter_destroyed`,
  `player_died/respawned`, `night_survived` и т.д.).
- `CraftSystem` — рецепты крафта.
- `DebugCapture` (`scripts/debug_capture.gd`) — прогон-тесты + автоскриншот по `-- --capture`.

## Ключевые системы и файлы
- **Игрок** `scripts/player.gd`: движение, стволы (`weapons`, авто-огонь у Автомата),
  топор (`swing_axe` — добыча/ремонт/ближний бой), ремонт только в упор, психздоровье
  (`_update_sanity`), смерть/возрождение (`_on_died`/`_respawn`/`_fly`), вьюмодел оружия
  (`_build_viewmodels`/`_update_viewmodel`), классовые способности (F).
- **Убежище** `scripts/shelter.gd` (на `ShelterPerimeter` в `main.tscn`) + `shelter_segment.gd`
  на каждой стене: общий HP на весь периметр, растёт с тиром (600 + 400/тир).
- **Дерево навыков** `scripts/skill_menu.gd` (UI, 5 рядов «ёлочкой», кривые Безье, глифы) +
  данные в `inventory_system.gd`. Класс выбирается на узле «мастерство».
- **HUD** `scripts/hud.gd` (панели ресурсов/статуса/витального, полосы HP/рассудка,
  полоса HP убежища по наведению/<90%, баннеры тревог, оверлей возрождения, виньетка).
- **Единый UI-стиль** `scripts/ui_style.gd` (`class_name UiStyle`) — тёмная тема, применяется к меню.
- **Меню** `build_menu.gd` (карточки построек с иконками/ценой/доступностью),
  `workshop_menu.gd` (крафт/тир/инструменты/C4; нехватка ресурсов — красным).
- **Постройки** `wall/turret/generator/storage/mortar_turret/gatling_turret/infirmary/campfire.gd`
  — HP + плашка Label3D над объектом (видна только при <90% HP).
- **Враги/волны** `zombie.gd`, `wave_manager.gd`, боссы/крикун/взрывной/джаггернаут.
- **Иконки** генерируются `tools/gen_font_icons.py` (Segoe UI Symbol) + `tools/gen_icons.py`.

## Текущее состояние
Готово: классы/навыки/боссы (4.x), полное дерево навыков «ёлочка» + иконки, психздоровье,
убежище-цель + возрождение, единый UI (HUD/меню/иконки монохром), видимые модели оружия,
авто-огонь, ремонт в упор, нехватка ресурсов в меню. Всё в `origin/main`.

Эффекты, помеченные «скоро» (`ready=false` в `SKILLS`, ждут своих систем): броня,
гранаты, тиры оружия, вышка связи, бетонный завод, ловушки, маскировка, умелый строитель.

## Рабочий процесс (кратко)
- Правка → capture-прогон (`-- --capture 3`) → чтение `debug/last_run.png` + консоли (EXIT=0)
  → коммит+push с русским сообщением. Подробности — в `.cursorrules`.
- `debug/` в .gitignore. Тест-сценарии — в `scripts/debug_capture.gd` (секция 6.999 показывает
  дерево навыков для финального снимка).
