# 🧟 ПЛАН ПРОЕКТА: «Убежище» (рабочее название)

> 3D survival + tower-defense, вдохновлён режимом «Убежище» из CS Nexon: Zombies.
> Движок: **Godot 4** · Разработчик: **соло** · Ассеты: **нейросети + бесплатные CC0** · Бюджет: **0**

## Легенда маркеров
- 🎯 — критично для **MVP** (первый играбельный билд)
- 🧱 — фундамент, делать рано (на этом стоит всё остальное)
- 🔁 — итеративно (возвращаешься много раз по ходу проекта)
- ⏳ — поздняя фаза (только ПОСЛЕ MVP)
- ⚠️ — высокий риск / частая причина провала / сложность
- 🤖 — здесь активно используем ИИ-генерацию

## ЭТАП 3 — Прототип основного цикла (MVP / Vertical Slice) 🎯⚠️

  - 3.2.4 Смерть + дроп ресурса
- **3.3 Спавн волн** 🎯

- **3.4 Ресурсы и крафт-минимум** 🎯

  - 3.4.2 Простой инвентарь (счётчики ресурсов)

# 🧭 КАК ЭТО ЧИТАТЬ (важно для соло-разработчика)

1. **Этапы 0–3 строго последовательны** — это путь до первого играбельного билда (MVP). Не перепрыгивай.
2. **Этапы 4–11 — НЕ линейны.** Это циклы, которые крутятся параллельно: добавил систему (4) → сделал ассеты (5) → прикрутил UI (6) и звук (7) → сбалансировал (9) → отполировал (10) → протестил (11) → повторил.
3. **Этап 12 (мультиплеер) и 14 (масштабирование)** — это и есть тот самый «потенциал на масштабный проект». Их откладываем сознательно.
4. **Граница MVP** = всё с маркером 🎯. Если что-то не 🎯 — оно ждёт, пока MVP не заработает.
5. ⚠️ **Главный враг проекта — не сложность, а объём.** 90% инди-игр умирают от scope creep. Anti-scope (1.6.3) — твой щит.
